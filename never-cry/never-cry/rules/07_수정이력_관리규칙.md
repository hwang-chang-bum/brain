# 수정 이력 관리 규칙

## 목적

절대울지마 집필 과정에서 초안이 흔들릴 때, 어떤 전개를 살리고 어떤 전개를 버렸는지 기록하기 위한 규칙이다.

---

## 이력 기록이 필요한 경우

아래 상황에서는 반드시 `history/` 문서에 기록한다.

- 사용자가 “설명형”이라고 지적한 경우
- 웹툰 콘티처럼 보인다고 지적한 경우
- 사건이 하린에게 돌아오지 않는다고 판단한 경우
- 새 단서가 기능하지 않는 경우
- 기존 설정과 충돌 가능성이 있는 경우
- 폐기한 아이디어를 나중에 재사용할 가능성이 있는 경우

---

## 기록 형식

각 폐기/수정 이력은 아래 형식을 따른다.

```md
## 제목

### 내용
초안 또는 전개 아이디어 요약

### 문제
왜 실패했는지

### 폐기/수정 이유
현재 구간의 기능과 어떻게 맞지 않았는지

### 재사용 가능성
완전 폐기인지, 일부 재사용 가능한지

### 살릴 요소
단어, 장면, 물건, 감정, 구조 중 남길 것

### 금지 요소
다시 반복하면 안 되는 것
```

---

## 수정 판단 기준

좋은 수정은 아래 조건을 만족해야 한다.

1. 사건이 먼저 발생한다.
2. 단서가 인물의 선택을 바꾼다.
3. 설명보다 장면의 압력이 커진다.
4. 윤하린이 사건 안쪽으로 한 걸음 더 밀린다.
5. 다음 화로 넘어갈 이유가 생긴다.

---

## 나쁜 수정

나쁜 수정은 아래와 같다.

- 설정을 더 많이 설명한다.
- 대사를 늘려 의미를 정리한다.
- 팀원을 추가해 회의 장면으로 만든다.
- 떡밥 단어만 더 강하게 던진다.
- 하린의 감정을 직접 설명한다.
- 과거 자료를 더 자세히 보여준다.

---

## 핵심 문장

> 수정은 정보를 추가하는 일이 아니라, 장면의 기능을 되찾는 일이다.
