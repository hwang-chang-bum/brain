# 떡밥 관리표

## 목적

이 문서는 절대울지마의 미스터리 단서를 관리하기 위한 표이다.

떡밥은 많이 던지는 것이 아니라, 반복될수록 의미가 좁혀져야 한다.
단서는 사건 속에서 나오고, 설명은 뒤로 미룬다.

## 공개 단계

- 단계 1: 독자가 이상하다고 느낀다.
- 단계 2: 인물이 이상하다고 인지한다.
- 단계 3: 기존 사건과 연결된다.
- 단계 4: 과거 사건의 일부와 맞물린다.
- 단계 5: 진실에 가까운 선택으로 이어진다.

## 핵심 떡밥

| 떡밥 | 현재 의미 | 숨은 의미 | 공개 상태 | 주의 |
|---|---|---|---|---|
| 괜찮지 않으면 | 끝나지 않은 문장 | 강도현이 하린에게 남긴 핵심 문장 | 일부 공개 | 뒤 문장을 너무 빨리 공개 금지 |
| D-17 | 자동 보조 태그 | 보호자 개입/기억 분리 절차 코드 | 일부 공개 | 감정 키 태그로만 취급 금지 |
| 보호자 유도 발화 | 현재 사건의 보호자 발화 | 하린 과거 보호자 관계와 연결 | 일부 공개 | 보호자=악인 단정 금지 |
| 분리 보류 | 오래된 절차 문구 | 하린 기억이 완전히 지워지지 않은 상태 가능성 | 암시 | 확정 금지 |
| 분석자 일치 | 하린에게 지정 배정된 이유 | 하린 반응값이 사건 개방 조건일 가능성 | 일부 공개 | 시스템이 마법처럼 보이면 안 됨 |
| 본인 반응값 대조 | 하린의 반응을 기준값으로 사용 | 하린이 과거 절차의 대상자였을 가능성 | 일부 공개 | 직접 폭로 금지 |
| G.D.H. | 외부 개입자 코드 | 강도현 암시 | 암시 | 이름 확정 금지 |
| 세 손가락 | 사진/손동작 | 하린의 유년 기억 조각 | 보조 암시 | 싸구려 반복 숫자/손동작 떡밥 주의 |
| 지정 배정 | 사건이 하린에게 배정됨 | 누군가 하린을 사건 안으로 끌어들임 | 공개 | 편의적 전개 금지 |
| 말해 | 현재 사건의 발화 | 강요/부탁/보호 사이의 애매함 | 공개 | 대사 하나로 의미 설명 금지 |

## 13~30화에서 금지되는 공개

- 방주의 정체 공개 금지.
- 노아 정체 공개 금지.
- 외계 문명 정체 공개 금지.
- RETURN 최종 진실 공개 금지.
- 강도현이 무엇을 했는지 확정 금지.
- 윤하린이 정확히 어떤 절차 대상이었는지 확정 금지.
- 한서연의 제3의 선택을 설명식으로 공개 금지.

## 좋은 떡밥 사용 방식

현재 사건 속에서 단서가 먼저 나온다.

예시:

- 통화 속 “말해”
- 시스템의 D-17 자동 태그
- 현행 분류표에 없는 구형 코드
- 문장의 일부만 출력
- 어른들이 단어를 듣고 반응
- 보고서에는 남길 수 없는 말이 등장

## 나쁜 떡밥 사용 방식

- 마지막 줄에 숫자만 던진다.
- 자료실에서 파일을 열자 진실이 나온다.
- 누군가 친절하게 과거를 설명한다.
- 하린이 혼자 떠올리고 혼자 깨닫는다.
- RETURN, 강도현, 노아, 방주 같은 큰 단어를 너무 자주 부른다.
- 떡밥이 사건 해결과 연결되지 않는다.
