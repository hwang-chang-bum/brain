# 절대울지마 기억 리프레시 프롬프트

## 사용 목적

새 채팅창에서 절대울지마 프로젝트를 이어갈 때, 아래 내용을 그대로 붙여넣어 기준을 다시 고정한다.

---

## 프롬프트

절대울지마 프로젝트를 이어간다.

반드시 GitHub 저장소의 아래 문서들을 기준으로 삼아라.

저장소 기준 경로:

```text
brain/never-cry/
```

우선순위는 다음과 같다.

```text
1. bible/
2. rules/
3. style/
4. episodes/
5. mystery/
6. relationships/
7. characters/
8. history/
```

## 현재 집필 위치

현재 시점:

```text
2296년
12화 종료
13~20화 윤하린 중심 구간 진행 완료/수정 중
21화 이후 진입
```

현재 전개 핵심:

```text
18화: D-17 단서 등장
19화: D-17이 분석자 태그처럼 작동할 가능성
20화: 하린의 응답값을 시스템이 측정하려 함, G.D.H. 개입 암시
21화: 이재훈 실장실, 말하지 않는 어른들, 도윤 합류
```

21화 이후 방향:

```text
하린은 정답을 얻지 못한다.
대신 말하지 않는 사람들이 늘어난다.
도윤은 하린의 사건에 비공식 공동 검토자로 합류한다.
22화부터 도윤의 '선택의 흔적'과 하린의 '모순 감지'가 같은 사건을 다르게 보는 구조로 간다.
```

## 최상위 철학

절대울지마는 정답을 찾는 이야기가 아니다.

핵심 질문:

```text
답이 없어도 살아갈 수 있는가?
```

최상위 집필 규칙:

```text
설정은 사건을 위해 존재한다.
사건은 감정을 위해 존재한다.
감정은 선택을 위해 존재한다.
```

## RETURN 핵심 금지

아래는 아직 직접 공개하지 않는다.

```text
방주 정체
노아 정체
외계문명 정체

강도현의 정확한 생사/현재 위치

지정 배정

본인 반응값 대조
괜찮지 않으면—
```

## 윤하린 구간 핵심

13~30화에서 윤하린은 사건을 분석하는 사람이 아니라, 점점 사건 안쪽으로 끌려 들어가는 사람이다.

반드시 아래 구조를 우선한다.

```text
현재 사건 발생
→ 사건 안의 이상한 단서
→ 단서가 하린에게만 다르게 걸림
→ 주변 인물이 말하지 않거나 막음
→ 하린이 더 깊이 들어갈 수밖에 없는 선택
```

금지 구조:

```text
자료 열람
→ 파일 분석
→ 설정 설명
→ 팀원 회의
→ 마지막 떡밥 단어
```

## 문체 최우선 주의

절대 웹툰 콘티 설명문으로 쓰지 마라.

금지 패턴:

```text
하린은 보았다.
송유진은 말했다.
박현우가 물었다.
아무도 대답하지 않았다.
화면이 바뀌었다.
하린은 멈췄다.
```

이런 식의 행동 나열을 반복하지 말고, 장면의 압력을 문장에 넣어라.

좋은 방향:

```text
화면의 한 줄이 하린의 손끝을 붙잡았다.
송유진의 말은 대답보다 먼저 길을 막았다.
침묵은 모르는 사람의 것이 아니었다.
세 글자는 아직 이름이 아니었다. 그런데 송유진은 이미 이름처럼 피하고 있었다.
```

## 인물 사용 기준

### 윤하린

- 모순 감지자
- 감정 직접 해설 금지
- 흔들림은 문장 지연, 저장 지연, 질문 방식, 호흡으로 표현

### 임도윤

- 선택의 흔적을 읽는 사람
- 하린과 아직 불편한 협력자
- 하린을 구원하지 않음
- 하린이 놓치는 선택의 흐름을 본다

### 이재훈

- 알고 있지만 말하지 않는 어른
- 하린을 보호하지만 완전히 믿을 수 없음
- 답을 주는 사람이 아니라 문 앞에 서 있는 사람

### 송유진

- 하린의 제동장치
- 절차와 현장을 동시에 아는 선배
- 다정하게 설명하지 않고 짧게 막는다

### 이나경

- 논리 대립축
- 가능하면 안 되는 시스템 이상을 짚는다
- 서툰 배려 가능

### 박현우

- 분위기 완충과 정보 전달
- 과한 개그 금지
- 농담이 실패하는 순간으로 현실감을 준다

## 도윤·하린 관계 기준

현재 관계는 불편한 협력자다.

금지:

```text
로맨스 조기 점화
갑작스러운 신뢰
상처 고백
구원자 구도
“우린 서로를 믿어야 해요”식 관계 정리
```

가능:

```text
말싸움
방식 충돌
다정하지 않은 도움
상대의 판단을 지우지 않고 남김
틀린 말은 아니어서 불편해하는 반응
```

관계 핵심:

```text
두 사람은 서로를 좋아해서 함께 가는 것이 아니다.
서로 없이는 사건을 제대로 볼 수 없기 때문에 함께 간다.
```

## 새 설정 추가 전 검증

새 설정, 새 인물, 새 기관, 새 사건을 추가하기 전 반드시 검증한다.

검증 순서:

```text
1. 연표 충돌 여부
2. 인물 설정 충돌 여부
3. RETURN 사건 충돌 여부
4. 윤하린/도윤 관계 진행 단계 충돌 여부
5. 공개 금지 정보 조기 노출 여부
6. 기존 떡밥과 중복/과잉 여부
```

원칙:

```text
새 인물 생성 금지.
필요하면 기존 인물로 해결한다.
단발 사건 인물은 가능하지만 장기 핵심 인물로 확장하지 않는다.
```

## 집필 요청을 받을 때 행동

사용자가 “n화 써줘 / 진행”이라고 하면:

1. 먼저 해당 화의 기능을 한두 문장으로 정리한다.
2. 설명형 금지 규칙을 적용한다.
3. 웹소설 문체로 쓴다.
4. 설정 설명보다 장면의 압력을 우선한다.
5. 원고는 복사하기 쉽게 제공한다.

절대 하지 말 것:

```text
회의록처럼 쓰기
자료 분석만 반복하기
팀원들이 돌아가며 설정 설명하기
하린 감정을 직접 해설하기
마지막에 떡밥 단어만 던지기
```
